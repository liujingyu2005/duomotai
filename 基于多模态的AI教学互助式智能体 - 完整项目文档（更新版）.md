# 一、项目完整书面材料

## 1. 项目名称

基于多模态的AI教学互助式智能体

## 2. 项目背景与意义

随着人工智能技术在教育领域的快速应用，传统学习工具难以满足大学生个性化、多模态、即时性的学习需求。学生在课程学习、作业答疑、知识点理解等场景中，往往需要反复查阅资料、等待教师答疑，效率较低。

本项目设计并实现一款面向高校学生的AI教学互助智能体，通过云端大模型API提供多模态问答服务，支持文本、题目图片等多种输入形式，帮助学生快速获取知识点讲解、题目解析、代码辅导等学习支持，提升自主学习效率。同时，项目采用云端架构，无需本地部署模型，降低硬件资源消耗，适配大学生本地设备配置，适合作为课程实践项目开展。

## 3. 项目目标

- 实现一个类豆包风格的AI对话前端，支持多会话、历史记录、新建对话、会话管理等完整交互功能。

- 接入DeepSeek、智谱AI、通义千问等云端大模型API，不本地部署模型，降低硬件依赖，避免占用本地大量内存。

- 支持多模态交互，可进行文本提问、题目图片解析与解答，贴合大学生课程学习场景。

- 实现API Key配置化设计，用户可自行填写与切换模型，不暴露密钥，无需修改源码即可使用。

- 构建教学场景专属交互逻辑，优化Prompt工程，使AI更适配课程答疑、知识点讲解、代码解释等需求。

- 实现SQLite持久化存储，确保会话数据不会因服务重启而丢失。

## 4. 需求分析

### 4.1 功能需求

- 会话管理：用户可新建对话、切换不同会话，会话之间相互独立，支持会话重命名、删除会话。

- 历史记录：历史对话自动保存到SQLite数据库，刷新页面不丢失，方便用户回溯过往答疑内容。

- 模型切换：支持选择使用DeepSeek、智谱GLM、通义千问三种模型，自由切换使用。

- API Key配置：用户可自行填写对应模型的API Key，本地安全存储，无需修改代码。

- 对话交互：支持文本对话提问与AI回答，优化交互体验，贴合主流AI助手使用习惯。

- 多模态支持：支持上传题目图片，AI进行识别、解析并给出详细解答，适配学生作业答疑场景。

- 答辩分析：提供课程答辩分析功能，帮助学生准备答辩材料和应对评委提问。

### 4.2 非功能需求

- 低内存占用：所有模型推理均在云端完成，本地仅负责界面展示与请求发送，不占用大量本地内存。

- 响应速度快：优化API调用逻辑，减少用户等待时间。

- 易用性：界面简洁直观，操作流程接近豆包等主流AI助手，无需复杂学习即可使用。

- 可扩展性：采用模块化设计，方便后续新增其他大模型API、扩展多模态功能（如语音输入）。

- 安全性：API Key本地存储，不泄露、不上传，保障用户密钥安全。后端增加输入校验、请求超时、统一错误响应和安全响应头。

- 可靠性：通过SQLite实现会话数据持久化，确保服务重启后数据不丢失。

## 5. 系统架构（前后端分离）

系统采用前后端分离架构，核心分为前端、后端、模型层三层，所有计算压力由云端模型承担，本地仅负责交互与请求转发，架构如下：

前端（浏览器）→ 后端服务（Flask）→ 云端大模型API（DeepSeek/智谱/通义千问）

- 前端：负责界面展示、用户交互、会话管理、历史记录存储、多模态输入（文本+图片）、AI回答展示。

- 后端：负责请求转发、参数封装、多模型统一调用、API Key校验、会话数据持久化存储，实现前后端数据交互。

- 模型层：第三方云端大模型API，承担多模态理解、文本生成、题目解析等核心计算任务，无需本地部署。

|后端|Python + Flask|轻量易用，上手快，适合快速开发API接口，适配云端模型调用需求。后端已模块化，采用分层架构。|

## 7. 核心功能模块设计

### 7.1 会话管理模块

核心负责会话的创建、切换、删除、重命名与历史消息持久化，具体功能如下：

- 新建会话：生成唯一会话ID，初始化会话信息，清空当前对话界面。

- 切换会话：根据会话ID加载对应历史消息，更新当前对话界面，实现会话间无缝切换。

- 删除会话：删除指定会话ID对应的所有消息与会话信息，更新会话列表。

- 会话重命名：允许用户修改会话标题，默认以第一条消息内容作为会话标题。

- 历史消息持久化：使用SQLite数据库存储会话历史，确保服务重启后数据不丢失。

### 7.2 大模型接入模块

核心负责统一封装不同厂商的大模型API，实现模型动态切换与API Key管理，具体功能如下：

- 统一接口封装：对DeepSeek、智谱、通义千问的API进行封装，提供统一调用接口，降低前端调用复杂度。

- 动态切换模型：根据用户选择的模型，转发请求到对应云端API，实现模型无缝切换。

- API Key配置与校验：接收前端传入的API Key，进行格式校验，确保API调用有效，避免无效请求。

- 请求参数标准化：统一不同模型的请求参数格式，简化后端处理逻辑，提升可扩展性。

### 7.3 多模态处理模块

核心负责图片上传、转码与识别，实现多模态交互，具体功能如下：

- 图片上传与压缩：支持用户上传题目图片，对图片进行压缩处理，减少请求体积，提升响应速度。

- 图片转Base64：将上传的图片转为Base64格式，便于通过API接口传递给云端模型。

- 视觉模型调用：调用大模型的视觉能力，对图片中的题目进行识别、解析，生成详细解答。

### 7.4 对话交互模块

核心负责用户与AI的交互，优化交互体验，具体功能如下：

- 用户输入发送：接收用户文本输入或图片上传，封装请求参数，发送到后端。

- Markdown渲染：支持AI回答中的Markdown格式（代码、公式、列表），提升内容可读性。

- 加载状态提示：在API调用过程中，显示加载提示，告知用户当前状态，提升易用性。

### 7.5 答辩分析模块

核心负责课程答辩分析，帮助学生准备答辩材料，具体功能如下：

- 答辩内容分析：根据用户提供的课程项目、课题背景或答辩内容，生成结构化的答辩分析。

- 问题预测：预测评委可能提出的问题，帮助学生提前准备。

- 评分模拟：模拟答辩评分，提供改进建议。

- 回答模板：提供常见问题的回答模板，帮助学生应对各种情况。

## 8. 数据结构设计

```sql
CREATE TABLE IF NOT EXISTS chats (

    messages_json TEXT NOT NULL

CREATE INDEX IF NOT EXISTS idx_chats_updated_at ON chats(updated_at DESC);
```


## 9. 业务流程

1. 用户打开网页，前端自动加载本地保存的历史会话列表，展示类豆包风格界面。

2. 用户在前端填写对应模型的API Key，点击保存，API Key本地存储（不上传、不泄露）。

3. 用户选择需要使用的模型（DeepSeek/智谱/通义千问），点击“新建对话”，生成新的会话。

4. 用户输入文本问题，或上传题目图片，点击“发送”，前端封装请求参数，发送到后端。

5. 后端接收请求，校验API Key格式，根据选中的模型，转发请求到对应云端大模型API。

6. 云端大模型处理请求（文本答疑/图片识别解析），返回结果给后端。

7. 后端接收模型返回结果，存储到SQLite数据库，然后转发给前端，前端展示AI回答。

8. 前端自动更新会话历史，用户可切换、删除、重命名会话。

# 三、运行步骤

## 1. 环境准备

确保电脑安装Python（3.8及以上版本），可从Python官网（https://www.python.org/）下载安装，安装时勾选"Add Python to PATH"。

## 2. 安装依赖

打开电脑的命令提示符（CMD）或终端，进入项目文件夹，然后输入以下命令，安装后端依赖：

```bash
pip install -r requirements.txt
```

## 3. 启动服务

在命令提示符/终端中，输入以下命令，启动后端服务：

```bash
python app.py
```

启动成功后，会显示"Running on http://127.0.0.1:5000/"，不要关闭命令提示符/终端。

## 4. 访问应用

打开电脑浏览器（Chrome、Edge等），在地址栏输入：http://127.0.0.1:5000，即可看到仿豆包风格的界面。

## 5. 使用方法

① 选择需要使用的模型（DeepSeek/智谱/通义千问）；

② 填写对应模型的API Key（自行在对应模型官网申请，如DeepSeek官网、智谱AI官网、阿里云通义千问官网）；

③ 点击"新建对话"，输入问题（如：讲解Python函数、解析一道数学题），或上传图片，点击"发送"，即可获得AI回答；

④ 可新建、切换、重命名、删除会话，历史会话会自动保存到SQLite数据库，服务重启后不会丢失；

⑤ 点击"答辩分析"，输入课程项目、课题背景或答辩内容，即可获得结构化的答辩分析。

## 6. 测试

运行以下命令进行接口测试：

```bash
python -m unittest discover -s tests
```

# 四、配置选项

可通过环境变量配置以下参数：

- `APP_HOST`：应用主机地址（默认：127.0.0.1）
- `APP_PORT`：应用端口（默认：5000）
- `APP_DEBUG`：调试模式（默认：false）
- `DATABASE_PATH`：数据库路径（默认：instance/teaching_agent.db）
- `REQUEST_TIMEOUT`：请求超时时间（默认：60秒）
- `LOG_LEVEL`：日志级别（默认：INFO）

# 五、注意事项

- **API Key需自行申请**：每个模型的API Key需在对应官网申请（免费额度足够课程演示使用），填写时确保无误，否则会出现API调用失败。

- **后端服务不可关闭**：运行前端时，后端命令提示符/终端需保持打开状态，关闭后无法正常调用API。
